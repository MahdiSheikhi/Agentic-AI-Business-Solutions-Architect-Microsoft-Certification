2 · Generative vs classic orchestration — same agent, two brains

The single most-tested Copilot Studio distinction. Learn each row as its own fact.

Topic selectiongenerative: by topic description (“the agent chooses”) · classic: by trigger phrases (NLU match, 5–20 per topic).
Tool callinggenerative: orchestrator calls tools directly · classic: tools only from within topics.
Knowledge usegenerative: searched proactively as part of the plan · classic: fallback via Conversational boosting when no topic matches.
Multi-intentgenerative combines topics + tools + knowledge + agents in one plan · classic runs one topic at a time.
Filling inputsgenerative: slot filling asks for missing inputs automatically · classic: authored Question nodes.
Generative is required forevent triggers · MCP · deep reasoning · computer use · connected agents.
Choose classic whenflows must be fixed and auditable: identity verification, payment capture, consent, regulatory wording, DTMF menus. Admins can disable gen-AI per environment.

3 · The language-understanding ladder

Direct exam bullet: standard NLP vs conversational language understanding vs generative orchestration. Four rungs, four trigger phrases.

Standard NLUbuilt-in trigger phrases with RegEx/list entities — the zero-infrastructure default for classic agents.
NLU+enhanced precision at enterprise scale, precompiled model requiring training, also optimizes speech recognition for voice — and needs a Dynamics 365 Contact Center license.
CLUbring a trained Azure AI Language model: intents → topic triggers, CLU entities appear in the agent. Requires classic orchestration, matching region, intents for all system topics; enabling deletes existing trigger phrases; retires 2029.
Generative orchestrationthe LLM plans every turn — choose for variable phrasing, multi-intent, knowledge-heavy self-service.
💡 Hook: “industry vocabulary, more languages, advanced entities” → CLU · “voice precision at enterprise scale” → NLU+ · “stop maintaining trigger phrases” → generative.

4 · Topics & the fallback pipeline

Direct exam bullet: “design topics, including fallback.” Follow one unrecognized utterance through the system.

Topic anatomytrigger + nodes (Message, Question, Condition, tool calls, Redirect); topics accept input/output variables; redirects map values and return to the caller.
Generative bonuswith generative orchestration, topic inputs auto-fill from conversation context and outputs can feed the generated response.
System topics(disable yes, delete no): Conversation Start · Conversational boosting · End of Conversation (resolution + CSAT) · Escalate · Fallback · Multiple Topics Matched · On Error · Reset Conversation · Sign in.
Fallback flowrephrase up to two times → redirect to Escalate; use UnrecognizedTriggerPhrase to pass the raw utterance to a flow/skill/external NLP.
Enrichment loopreview what hit Fallback and boosting, then promote recurring questions into authored topics.

Agent flows & prompt actions

6 · Agent flows vs Power Automate cloud flows

Direct exam bullet: “design agents and agent flows.” The billing and the calling convention are the tested edges.

Agent flows

  • Built in Copilot Studio (designer or natural language)
  • 13 Copilot Credits / 100 actions — no Power Automate license
  • Natively callable by the agent; high throughput, low latency
  • Human-in-the-loop approvals — incl. multistage & AI approvals; premium connectors included
  • Can't be copied/shared/co-owned; desktop flows not callable

Cloud flows

  • Built in Power Automate; own per-user/per-flow licensing
  • Callable only as a solution flow with trigger “When an agent calls the flow” + action “Respond to the agent”
  • Broad standalone automation; desktop flows (RPA)
  • Conversion is one-way: cloud flow → agent flow
Billing edgeagent flows cost 13 credits per 100 actions; when prepaid credits run out, new runs are blocked immediately (in-progress runs finish; the agent keeps chatting).
Calling conventiona cloud flow is agent-callable only as a solution flow with the “When an agent calls the flow” trigger and “Respond to the agent” response.
When to use agent flowsdeterministic multistep actions the agent triggers — record updates, notifications, approvals; a newer “Workflows” format is in preview.
One-way streetcloud flow → agent flow conversion exists; there is no path back.

7 · Prompt actions — anatomy of a reusable prompt

Direct exam bullet. A prompt is a single-turn model instruction reusable as an agent tool, topic node, or flow node.

Prompt structureinstructions + input variables (text, image, document) + optional Dataverse grounding. Prompts in no-auth agents can't use Dataverse knowledge.
Bring-your-own modelvia the Azure AI Foundry connector: deployment name, base model name, chat-completions endpoint; GPT-5-family/Responses-endpoint models unsupported; governable by DLP.
Output formatsText or JSON (auto-detected or custom fixed schema — format locks at save time); agent-flow prompt nodes add Document output (Word from template).
Test & evaluate“Test prompt” shows projected credit/token consumption; prompt evaluations (preview) score a test suite across prompt versions.
BillingAI-tools meters: basic 1 · standard 15 · premium 100.
Prompt actions for M365 Copilotwizard order: action details (name/description drive semantic matching) → create prompt → input/output parameters → test in M365 Copilot → publish.
Authentication designuser auth = permission-trimmed actions per signed-in user; maker connection = fixed identity. Agent sign-in: Authenticate with Microsoft (default, required for Teams/M365) · manually (OAuth2, exposes User.AccessToken) · none (public; DLP can block).
💡 Hook: “action must run as each signed-in user” (audit, row-level security) → tool uses user authentication and the agent is not “No authentication.”

9 · Grounding data quality — the five dimensions

Direct exam bullet: “design data processing for AI models and grounding.” Each dimension has a design response.

Accuracyfix inconsistencies and duplicates before indexing; standardize schemas.
Relevanceanticipate real user queries; make fields searchable/filterable only where needed — index bloat hurts relevance and cost.
Timelinessfreshness = window between source change and index availability, tracked against SLOs; “an index should only be as fresh as required.”
Cleanlinessremove noisy, boilerplate, junk content; curate before you index.
Availabilityexternalize grounding data into a search index instead of querying source systems live — protects sources, keeps answers available during outages.
Core processing patternschunking vs full-document (accuracy vs token cost/latency) · embeddings + hybrid search (vector + keyword + semantic ranking) · security trimming via filterable identity fields · side-by-side index deployment for safe updates and emergency removal.
💡 Hook: when an agent answers wrongly, debug in order: (1) data accurate/clean? (2) right source connected & authenticated? (3) user permitted? (4) synonyms/glossary needed?

11 · AI in canvas apps · generative pages · the agent feed

Three direct exam bullets in one section — the modern pattern for each, and the deprecated trap.

Copilot control — the trapchat over the app's Dataverse data, but it cannot be added to new canvas apps from Feb 2, 2026 — never the modern answer.
Embed a Copilot Studio agentthe modern in-app assistant — works on web and mobile.
Prompts as Power Fx functionsadd the prompt as a data source → 'PromptName'.Predict(input) → bind result.text — the primary pattern for generative steps (classify, extract, draft) inside a canvas business process.
Model-driven counterpartsapp Copilot chat, row summaries, form fill assistance, NL view filtering.
Generative pages(model-driven; GA Oct 30, 2025): describe a page in NL (+ up to 6 Dataverse tables, reference images) → generates React/TypeScript, refined conversationally; solution-aware.
Code-first generative pagesbuild with external AI tools (GitHub Copilot CLI, Claude Code) locally against Dataverse, push via CLI — propose for pro-dev teams, unsupported regions, or pixel-level UX without full PCF development. Makers must validate generated code.
Agent feed for appssurfaces agent-generated actionable tasks in model-driven apps — the human-supervision surface for autonomous agents. Sections: Needs attention (request_assistance, invoke_data_entry) and Completed (request_review).
Agent feed prerequisitesagents must be onboarded to the Power Apps MCP server (only MCP-server agents appear from May 1, 2026); everyone with access to the Agent Task table sees feed items.
💡 Hooks: “include AI in a canvas-app process” → prompt as Power Fx function (or embed an agent) · “AI-generated custom page UX, pro-dev, worldwide” → code-first generative pages · “humans must supervise what autonomous agents did to records” → agent feed.

Direct exam bullet: apply PPWAF to intelligent application workloads.

The unique pillarExperience Optimization — usability and relevance, including conversational UX design for agents. The other four align to Azure WAF.
