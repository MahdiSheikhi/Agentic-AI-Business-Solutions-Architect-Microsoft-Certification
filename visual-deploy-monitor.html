Instruments — recommend the process & tools

1 · The monitoring tool map — five layers, five instruments

Every monitoring question names a layer; every layer has exactly one right instrument. Learn the pairs.

Copilot Studio analyticsPer-agent KPI home: sessions, engagement, resolution, escalation, CSAT. Retained 360 days, timestamps UTC, test-panel traffic excluded.
Application InsightsConnect by pasting the connection string in Settings → Advanced. Streams messages/events, topic triggers, node execution; options to log conversation details & sensitive properties.
KQL over customEventsThe documented way to find ContentFiltered (content-filter block) events in an agent's telemetry — query customEvents in App Insights.
Foundry observabilityThree pillars: tracing · agent monitoring dashboard · continuous evaluation (each gets its own figure below).
PPAC Licensing → Copilot StudioCredit burn billed vs non-billed by feature, per environment and per agent — where you check burn rate and set per-agent caps.
Agent Insights Hub (Power CAT Kit)One dashboard merging App Insights + transcripts + PPAC credit usage — the "single pane of glass" answer.

Metrics — monitor agent performance

3 · The KPI funnel — definitions, exactly

Every KPI is a percentage of the stage above it. The exam tests the denominators and the resolved/escalated sub-definitions.

KPIExact definition
Total sessionsCount of analytics sessions — a session caps at 60 minutes / 100 turns, so one conversation can produce several sessions
Engagement rate% of sessions that trigger a custom topic or enter Conversational boosting / Escalate / Fallback (generative mode: the plan includes a non-system topic, knowledge, or tool)
Resolution rate% of engaged sessions resolved — confirmed (user answers “yes” to “Did that answer your question?”) or implied (ends without escalation and logic implies success)
Escalation rate% of engaged sessions escalated; three logged reasons: System intended · System unintended · User requested
Abandon rateEngaged sessions ending unresolved and unescalated after the 60-minute window closes
CSATAverage of the 1–5 End-of-Conversation survey
💡 Hook: engagement and resolution use different denominators — resolution/escalation/abandon are all “% of engaged”, never “% of total”. Resolved-implied exists: no “yes” needed.

4 · The retention clock — 28 · 30 · 89 · 360

Four retention windows, one chant. Anchor each number to what it holds.

  • 28 days — session details and Reactions (thumbs up/down + comments) live in-product this long. Reactions are not available on the M365 Copilot channel.
  • ~30 days — transcripts sit in the Dataverse ConversationTranscript table (default bulk-delete; extendable). Long-term custom analytics = Azure Synapse Link → ADLS Gen2.
  • 360 days — the analytics dashboards themselves. Timestamps are UTC; test-panel chats never appear.
  • Reading transcripts requires the Bot Transcript Viewer role — CSAT drill-downs and thumbs-down investigations start here.

5 · Three analytics views — the page adapts to the agent

Conversational, autonomous, or hybrid — Copilot Studio switches its analytics layout to match how the agent works.

Conversational viewThe funnel above: sessions → engaged → resolved / escalated / abandoned + CSAT. For chat-first agents.
Autonomous viewRun outcomes, trigger/tool/knowledge usage, and Savings estimates (time/money the runs saved). For trigger-driven agents there is no “conversation” to grade.
Hybrid viewBoth sets side-by-side when an agent chats and runs autonomously.

AI-based analysis — the tools that find issues for you

6 · The feedback toolbox — AI analyzing your users' feedback

The “apply AI-based tools to analyze and identify issues” bullet maps to these five in-product features.

ThemesAI-clusters user questions into topics — and converts a theme into an evaluation test set in one click. Production telemetry becomes regression tests.
Generated answer rate & qualityAI labels sampled answers Good / Poor against four checks: completeness, relevance, groundedness, abstention.
AI-generated insights (preview)Daily recommendations on coverage gaps — which questions the agent keeps failing and what knowledge to add.
Custom metricsDefine up to three natural-language business metrics and let AI score conversations against them.
ReactionsThumbs up/down + free-text comments; 28-day retention; feeds the backlog. Not on the M365 Copilot channel.

7 · Foundry observability — the three pillars

Trace it, watch it, evaluate it forever. Each pillar answers a different production question.

  • Tracing follows OpenTelemetry GenAI conventions into App Insights — a span per model call, tool invocation, agent decision. Supported for Agent Framework, LangChain/LangGraph, and the OpenAI Agents SDK.
  • The agent monitoring dashboard (preview) unifies token usage, latency, run success rate, evaluation scores and red-team results in one place.
  • Continuous evaluation samples production responses (rate-limited ~100/hour by default), plus scheduled evaluations against benchmarks and scheduled red-team scans; alerts fire on latency / tokens / eval scores.
  • Evaluation run comparison applies statistical significance — verdicts Improved / Degraded / Inconclusive. This is the documented way to justify a model swap.

8 · The alert thresholds — two numbers to investigate

Foundry's dashboard documents two “go look now” trip-wires. They appear verbatim in questions.

> 10 slatency → investigate the bottleneck span
< 95%run success rate → investigate failed runs
~100/hrcontinuous-evaluation sampling rate (default)
💡 Hook: latency > 10 s → trace the span, shorten prompts, check throttling. Success < 95% → read the failed run details. Neither says “swap the model” — that needs an evaluation run comparison first.

Tuning — interpret telemetry, then act

9 · The signal → action matrix (the exam's favorite table)

A scenario names a symptom; you pick the architect's move. Read each row until the pairing is reflex.

Telemetry signalArchitect's action
Low engagement rateCoverage gap or unclear entry points → add topics / trigger phrases, clarify the agent's description & greeting
Unanswered questions / low generated-answer rateAdd or expand knowledge sources; act on AI-generated insights
Answers labeled ungrounded / irrelevant / incompleteTune knowledge content, instructions, topic config; consider stricter grounding (e.g., Bing Custom Search scoping)
Thumbs-down · low CSAT · negative sentimentDrill into transcripts (Bot Transcript Viewer); fix the offending topic/knowledge; prioritize by Themes volume
High escalation — “System unintended”Users are getting stuck → fix topic flows; note Omnichannel deflection analytics compute differently — don't compare raw numbers
Knowledge-source errorsRepair or replace the failing sources (source/tool error panels show which)
Content-filter blocksKQL for ContentFiltered events + transcripts → adjust the moderation level or user guidance
Foundry: slow / token-hungry / failing runsTrace the bottleneck span · shorten prompts · check throttling · investigate failed runs
Everything prompt-tuning can't fixCompare models via evaluations → swap model or fine-tune, validated by run comparison with statistical significance

10 · The continuous-improvement loop — feedback becomes backlog becomes tests

“Analyze backlog and user feedback” is a cycle, not a report. Four stages, one monthly ritual, one golden rule.

  • The documented loop: assess usage → define outcomes → analytics strategy → data-driven optimization, repeated on a cadence.
  • Monthly optimization review agenda: deflection, resolution, low-resolution topics, unrecognized utterances, per-channel performance.
  • Feed Reactions + CSAT + Themes + unanswered-question lists into a prioritized backlog — feedback is triaged like product work, not read like fan mail.
  • Golden rule: every fix becomes a regression test set (Themes → test set in one click) so the same failure can never return unnoticed.
💡 Final hook: the chant for this whole page — 28 · 30 · 89 · 360 retention, 10 s · 95% · 100/hr thresholds, and “resolution is a % of engaged”.
