2 · The six CAF AI-adoption phases

Three run in order, three never stop. Know which artifact belongs to which phase.

💡 Hooks: maturity L1 = ready-made Copilots · L3 = RAG apps in Foundry · L4 = training custom models on AKS/VMs. Copilots reach ROI in days–weeks; custom Azure AI in weeks–months. Governance enforcement = Azure Policy + Purview; model onboarding = sandbox first, then production catalog.

3 · The four technology adoption models

A staircase trading simplicity for customization. The exam gives you a phrase — you name the step.

💡 Hook: the four exam trigger phrases map 1:1 — “fastest ROI, minimal customization” → SaaS Copilot · “low-code department automation, SaaS governance” → Copilot Studio · “model choice / pro-code / evaluations” → Foundry · “bring-your-own model, strictest isolation” → IaaS.

5 · Anatomy of an agent — the 5 core components (CAF)

Every agent question comes back to these five parts. The model reasons; the other four give it scope, facts, hands, and history.

💡 Hook (Copilot Studio flavor): the orchestrator picks topics/tools by their names and descriptions — descriptions are functional configuration, not documentation. An agent can't act on instructions that reference resources it doesn't have.

6 · The agent autonomy spectrum

More autonomy = more value and more governance. Exam questions often hinge on picking the right point on this line.

Before any of these: if the process is fixed and rules-based → Power Automate / code. If it's static Q&A over fixed documents → classic RAG. Agents earn their cost only on ambiguous, multi-step work.

7 · ROI & TCO — the two formulas and the discipline

The exam expects you to compute these and to know what makes a measurement honest.

ROI % = (benefit − cost) ÷ cost × 100classify benefits tangible vs intangible
Payback = cost ÷ monthly net benefitmonths until the investment repays itself

TCO components

  • hardware · software / licensing
  • maintenance · energy & cooling
  • personnel · downtime · replacement
  • vs the platform's consumption cost (credits / tokens / PTU)

Measurement discipline

  • baseline before rollout
  • keep a non-user comparison group
  • time saved = leading indicator — track where reclaimed time goes
  • avoid vanity metrics: seats purchased ≠ value
💡 Hooks: value measures = efficiency · revenue · productivity · TCO · satisfaction. Drivers = performance improvement · cost savings · risk mitigation. Build-vs-buy-vs-extend: prebuilt fits → buy/configure; unique process on standard platform → extend; differentiating IP → build. The Business Value Toolkit (CoE Starter Kit) structures the value story.

8 · The AI Center of Excellence

Build order, the four agentic functions, and the three anti-patterns the exam names.

The 4 agentic-CoE functions

  • Govern — release gates, audit logs
  • Enable — golden paths, templates, training, community
  • Optimize — quality / cost / value monitoring
  • Scale — a repeatable pipeline

The 3 anti-patterns

  • Gatekeeper — every decision bottlenecks through the CoE
  • Ghost — sets standards nobody sees enforced
  • Perfectionist — polishes forever, ships nothing
💡 Hooks: shift centralized → advisory when you see approval delays, knowledge bottlenecks, CoE-vs-team friction. Operating model: platform teams enforce guardrails · workload teams own agent lifecycles · the CoE advises. The Power Platform CoE Starter Kit stays the tenant-governance toolkit.

9 · Prompt anatomy — remember GCSE

Microsoft's canonical four-element prompt structure. The mnemonic is the answer to the exam question.

GGoalwhat output you want — “write a step-by-step cleaning procedure”
CContextwhy / for whom / background — “for SmartBrew machine owners”
SSourcewhich data, knowledge or plugins to use — “use the company knowledge base”
EExpectationsformat, tone, audience, length — “casual tone, numbered list, ≤10 steps”
💡 Prompt-library numbers: organizational prompts ≤ 1,000/tenant, bulk CSV import ≤ 100 prompts / 5 MB, pin 4 suggested prompts, ~3 h to propagate, per-language (no auto-translation). Copilot Prompt Gallery lives in the Substrate, inside the tenant boundary. Copilot Studio prompt = instructions + context (input variables + grounding data).

11 · Copilot Studio orchestration — generative vs classic

Same agent, two brains. The exam tests which behaviors belong to which mode — and which language-understanding option fits.

Generative (default for new agents)

  • topics chosen by description — “the agent chooses”
  • tools called directly by the orchestrator
  • knowledge searched proactively in the plan
  • multi-intent: combines topics + tools + knowledge + agents
  • generative slot filling asks for missing inputs
  • required for: event triggers, MCP, deep reasoning, computer use, connected agents

Classic (deterministic)

  • topics chosen by trigger phrases (NLU match, 5–20 per topic)
  • tools only from within topics
  • knowledge = fallback (Conversational boosting)
  • one topic at a time; authored Question nodes
  • choose for fixed, auditable flows: identity verification, payments, consent, regulatory wording, DTMF menus
💡 Language-understanding ladder: Standard NLU = built-in trigger phrases · NLU+ = enterprise precision + voice, needs a D365 Contact Center license · CLU = bring your Azure AI Language model — needs classic orchestration, matching region, and enabling it deletes existing trigger phrases (CLU retires 2029) · generative orchestration = the LLM plans, no trigger-phrase maintenance.

13 · Foundry — catalog split & deployment matrix

Two questions hide here: who supports the model, and where inference runs. Data at rest never moves — only processing does.

Sold directly by Azure (“Azure Direct”)

  • Microsoft-hosted · Azure SLAs · Microsoft Product Terms
  • all Azure OpenAI + selected DeepSeek / Meta / Mistral / xAI / Cohere
  • PTU fungible across Azure Direct families

Partner & community

  • provider-supported · “Non-Microsoft Product” terms
  • most of the catalog — incl. Anthropic Claude, Hugging Face
  • serverless/standard or managed compute

Data Zonewithin US / EU / APAC zonezone-level residency compliance — same three billing flavors
Regionalthe resource's region onlystrictest residency · lower quota
Developeranyno SLA — cheap evaluation of fine-tunes
💡 Hooks: data at rest always stays in the resource geography — deployment type changes only processing location. Spillover routes 429-overflow from provisioned → standard (Azure OpenAI models only). Model router: Balanced / Cost / Quality — its context window = the smallest underlying model; Claude targets deploy separately. Leaderboards rank quality · safety · cost · throughput. 15 · What a Copilot Credit buys

Credits per event, from cheapest to priciest. Hover a bar for what triggers the charge. Chant the ladder: 1 · 2 · 5 · 10.

Different units, so not on the chart: agent flows = 13 credits / 100 actions · AI tools = 1/15/100 · voice = 10/35/75 per minute. Capacity pack = 25,000 credits/month, tenant-pooled, no carryover, 125% overage grace (flows block instantly). M365 Copilot-licensed users: no charge on listed features — computer use excluded; for agent flows only the “When an agent calls the flow” trigger. Test-panel chats don't bill.

16 · Two ways to attack an agent — and the defense for each

UPIA comes from the User; XPIA hides in eXternal content. Different doors, different locks.

💡 Hooks: filters off / annotate-only needs Limited Access approval; protected-material detection is on by default. CMK revoked → no new training/deployments, but deployed models keep serving. Purview = CopilotInteraction audit + DSPM for AI posture; Defender for Cloud = runtime AI threat alerts.

17 · Agent identity — Entra Agent ID

Who is the agent when it knocks on a resource's door? Depends on whether a human is in the room.

💡 Hook: attended = OBO (“the user is the badge”) · autonomous = client credentials (“the agent brings its own badge”). Foundry RBAC: fine-tune training = Foundry User (data plane), but deploying a fine-tune is control-plane — Foundry Owner covers both.

18 · ALM — how an agent travels to production

Solutions carry the agent; pipelines move it; an evaluation gate decides if it ships.

💡 Hooks: a period in a topic name blocks export. Component collections = reusable topic/tool sets. Custom models = MLOps/GenAIOps + progressive exposure + drift monitoring. Foundry CI/CD bootstrap = azd pipeline config.

19 · Environment strategy — tiers and the governance pyramid

Route makers away from Default, govern in layers, and let environment groups do the enforcement at scale.

💡 Hooks: environment groups need Managed Environments, an environment joins one group, and group rules lock local settings. Default routing sends new makers to personal managed developer environments. Regions matter twice: data residency + gen-AI regional availability (missing capacity → per-environment cross-geo consent in PPAC, or features stay off). Third-party AI: DLP connector classes Business / Non-business / Blocked + endpoint filtering.

20 · The analytics funnel — KPI definitions, exactly

Deploy's beating heart. Every KPI is defined off the previous stage — learn the denominators.

CSAT = average of the 1–5 end-of-conversation survey · Foundry agent evaluators: Intent Resolution · Task Adherence · Tool Call Accuracy.

💡 Retention chant — 28 · 30 · 360: session details & reactions 28 d · Dataverse transcripts ~30 d default (extendable; Synapse Link → ADLS Gen2 for long-term) · analytics 360 d. App Insights = paste the connection string; KQL over customEvents finds ContentFiltered events; viewing transcripts needs the Bot Transcript Viewer role.

23 · The six Responsible AI principles

Standard v2 turns these into 14 goals with requirements — plus impact assessments and transparency notes. Memorize all six by name.

⚖ Fairnesstreat all groups equitably — test for bias across demographics
🛡 Reliability & safetyperform safely under expected and unexpected conditions
🔒 Privacy & securityprotect data and resist misuse — DLP, labels, least privilege
🤝 Inclusivenessempower everyone — accessibility is a design requirement
🔍 Transparencypeople know they're talking to AI — transparency notes, citations
👤 Accountabilityhumans stay answerable — named owners, oversight, audit trail

Shared responsibility: SaaS agents (M365 Copilot) → Microsoft owns most layers. PaaS agents (Foundry) → you own the system message, grounding, and content filters. Review = principle-by-principle checks + named owner + human oversight + audit trail + release-gated evals/red teaming.
