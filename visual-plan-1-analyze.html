A · Assess the use of agents

automation · analytics · decision-making

1 · The skill map — what "analyze requirements" tests

Three skills, one storyline: decide if an agent fits, check the data that will ground it, then make that data available to every AI system with governance intact.

1Assess agents for automation, analytics & decisions
2Review grounding data on 5 dimensions
3Organize data for other AI systems

An AI agent = software that uses a generative model as a reasoning engine to interpret inputs (user messages, events, other agents' messages), decide what to do, and act — unlike traditional automation that follows fixed rules. The architect's first job is deciding whether a use case needs an agent at all.

2 · Anatomy of an agent — the 5 core components (CAF)

The model reasons; the other four give it scope, facts, hands, and history. Every agent question comes back to these five parts.

1Generative model = the reasoning engine that interprets inputs and decides what to do at each step.
2Instructions define scope, rules and boundaries — an agent can't act on instructions that reference resources it doesn't have.
3Retrieval / knowledge provides grounding data — CAF: "access to relevant, high-quality data is critical for reducing hallucinations."
4Actions / tools = functions, APIs, connectors — the agent's hands on business systems.
5Memory holds conversation history and state so context survives across steps and sessions.

3 · The agent autonomy spectrum

Three agent types with rising autonomy, complexity — and governance duty. The exam gives a scenario; you place it on this line.

6Retrieval / productivity agents retrieve and synthesize information to speed decisions — Q&A, knowledge management.
7Task / action agents perform specific actions inside defined workflows — create a ticket, update a record.
8Autonomous agents manage multi-step processes with minimal oversight, using triggers to run, stop, escalate (e.g., supply-chain optimization) — they need the most rigorous governance.
9Copilot Studio's design vocabulary for the same ladder: prompt-and-response → task → autonomous agents.

4 · Do you even need an agent? — the CAF decision flow

Agents earn their cost only on ambiguous, multi-step work. Read top to bottom — first "yes" wins.

10Fixed, rules-based process → regular code, Power Automate, or non-generative AI — faster, cheaper, more reliable than an agent.
11Static Q&A over a fixed document set → classic RAG (FAQ bot), not an agent.
12Agents fit when workflows are ambiguous, inputs unstructured, and dynamic tool selection / reasoning is required.
13Agent vs RAG: RAG follows a deterministic retrieval pipeline; an agent lets the model choose knowledge/tools at each step — multi-step reasoning, but nondeterminism that demands testing and governance.

5 · Deterministic vs generative — agents and agent flows in Copilot Studio

Same product, two execution philosophies. Billing and licensing ride along.

Deterministic — topics & agent flows

  • same input → same output, auditable
  • agent flows billed via Copilot Credits — no Power Automate license needed
  • cloud flows need Power Automate licensing
  • cloud flow → agent flow conversion is one-way

Non-deterministic — generative orchestration

  • the model plans: picks topics, tools, knowledge
  • output can vary run to run
  • needs evaluation, testing, governance
  • enables multi-intent, dynamic reasoning
14Topics and agent flows are deterministic; agents with generative orchestration are non-deterministic.
15Agent flows bill in Copilot Credits (no Power Automate license); cloud flows require Power Automate licensing.
16Power Automate cloud flows can be converted into agent flows — but the conversion is one-way.

8 · The four data types of an AI workload (Well-Architected)

Grounding = supplementing the model at inference with data it wasn't trained on. First, know which data is which.

SSource dataproduction data — structured, semi-structured, unstructured; where grounding content originates
GGrounding datasupplied to the model at inference, combined with the user query into the prompt (usually via RAG)
BBehavioral instructionsstyle / tuning data that shapes how the model responds
UUser / turn datasession state — retain as briefly as possible
21Source data = raw production data in structured, semi-structured and unstructured forms.
22Grounding data is combined with the user query into the prompt at inference time — the heart of RAG.
23Behavioral instructions = style / tuning data that steers tone and behavior.
24User / turn data is session state — the guidance says retain it as briefly as possible.

9 · The RAG pipeline — and the chunking tradeoff

Seven stages between a raw document and a cited answer. The design decision the exam probes: chunks or full documents?

Full-document grounding

  • maximum answer accuracy / context
  • but high token cost and latency

Chunked grounding

  • scalable and cheap per query
  • but design-sensitive — bad chunking breaks answers
25RAG pipeline concerns: data preparation → chunking → embeddings/vector search → indexing → retrieval → reranking → citations.
26Tradeoff: full documents = accuracy but token cost/latency; chunks = scalable but design-sensitive.

10 · The five review dimensions — remember A·R·T·C·A

The exam's exact wording for grounding-data review. Picture the star: if any point collapses, answers degrade.

27Accuracy — authoritative, correct, tested for groundedness; metrics: groundedness, answer relevance, evidence relevance.
28Relevance — content must match the use case; adding unrelated content (e.g., to a conversational-boosting topic) causes misleading answers.
29Timeliness — test for stale data and version mismatches; connectors sync on schedules; real-time connectors avoid staleness in dynamic domains.
30Cleanliness — deduplicated, well-structured, chunk-friendly, no conflicting versions; noisy data raises hallucination risk.
31Availability — reachable and permissioned: agent-user auth means users only see what they can access; Dataverse search must be enabled; indexes must scale under load.
💡 Hook: A·R·T·C·A — Accurate facts, Relevant scope, Timely freshness, Clean structure, Available with permissions. The exam names these five words verbatim.

2Copilot Studio checksgrounding, provenance and semantic-similarity checks — and answers cite sources
3"Allow ungrounded responses"a setting that loosens citation restrictions — off = answers must derive from knowledge
32Groundedness detection (Foundry Tools / Azure AI Content Safety) can verify a response is supported by its source material.
33Copilot Studio performs grounding, provenance and semantic-similarity checks and cites sources in generative answers.
34The "Allow ungrounded responses" setting loosens citation restrictions — leave it off to force knowledge-derived answers.

15 · Microsoft 365 Copilot connectors — synced vs federated

Formerly Graph connectors. Two models with opposite philosophies: copy-and-index vs fetch-on-demand.

Synced connectorFederated connector
Mechanismingest + semantically index into the GraphMCP-based, query-time fetch
Data movementcontent copied into the semantic indexnone — data stays in the source
Items carrycontent + metadata + ACLs (permission-trimmed)read-only results, no semantic indexing
Freshnesscontinuous sync on a schedulealways live
Best forbroad, searchable, citation-friendly knowledgedynamic or regulated data that must stay in source
44Copilot connectors bring external/LOB content into the Graph semantic index so M365 Copilot, Microsoft Search, Copilot Studio agents and Researcher can use it.
45Synced = ingest + index; items carry content, metadata and ACLs with permission trimming and continuous sync.
46Federated = MCP-based query-time fetch, no data movement, read-only, no semantic indexing — for data that must stay in its source system.
47100+ prebuilt connectors (ServiceNow, Salesforce, Confluence…); custom ones via the Copilot connectors API: app registration → connection → schema → ingest items with ACLs.

16 · Copilot connectors vs Power Platform connectors as knowledge

Same word, different jobs — a classic exam distractor pair.

Copilot connectors

  • search-and-index model
  • broad, searchable, citation-friendly knowledge
  • content lands in the Graph semantic index

Power Platform connectors

  • live API bridges for real-time facts & transactions
  • usable as real-time knowledge in Copilot Studio (preview)
  • no indexing — every question hits the API
48Copilot connectors = search-and-index knowledge; Power Platform connectors = live API bridges for real-time facts and transactions (also usable as real-time knowledge in Copilot Studio, preview).
