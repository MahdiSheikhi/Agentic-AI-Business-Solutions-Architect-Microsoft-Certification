Copilot Studio II — Tools, Agent Flows & Prompt Actions

The tool palette

Add tools of these kinds: prebuilt connectors (1,000+), custom connectors, agent flows, prompts, REST APIs (import OpenAPI/Swagger), Model Context Protocol servers, computer use, plus Bot Framework skills (pro-code, synchronous, private endpoints). Per-tool design decisions: user vs maker authentication (user credentials → permission-trimmed actions; maker connection → acts with a fixed identity), tool descriptions (drive orchestrator selection), inputs AI-filled vs fixed at authoring time. DLP/connector policies govern every connector-based tool — including MCP.

Agent flows vs classic cloud flows (high-yield table)

Agent flows Power Automate cloud flows
Built/managed in Copilot Studio (designer or natural language) Power Automate
Licensing Copilot Credits — 13 per 100 actions; no Power Automate license Power Automate per-user/per-flow
Callable by agent Native Must be a solution flow with "When an agent calls the flow" trigger + "Respond to the agent" action
Extras High throughput, low latency; human-in-the-loop approvals (incl. multistage & AI approvals); premium connectors included Broad standalone automation, desktop flows (RPA)
Limits Can't be copied/shared/co-owned; desktop flows not callable —
Conversion One-way cloud flow → agent flow —
Enforcement New runs blocked immediately when prepaid credits are exhausted (in-progress runs finish; agent keeps chatting) Own licensing

Use agent flows for deterministic multistep actions an agent triggers (record updates, notifications, approvals). A newer "Workflows" format is in public preview.

Prompt actions (AI Builder prompt builder) — direct exam bullet

A prompt is a single-turn model instruction reusable as an agent tool, topic node, or flow node:

  • Structure: instructions + input variables (text, image, document) + optional grounding with Dataverse tables ("use your own data"). Prompts in no-auth agents can't use Dataverse knowledge.
  • Model & settings: pick the chat model — including bring-your-own Foundry model (via the Azure AI Foundry connector: needs deployment name, base model name, chat-completions endpoint; GPT-5-family/Responses-endpoint models not supported; governable by DLP) — plus temperature, code interpreter, reasoning.
  • Output formats: Text or JSON (auto-detected schema or custom fixed schema; format locks at save time); agent-flow prompt nodes add Document output (Word from template).
  • Test before save: "Test prompt" with sample data; projected credit/token consumption shown; prompt evaluations / accuracy scoring (preview) builds a scored test suite across prompt versions.
  • Billing: AI-tools meters (basic 1 / standard 15 / premium 100).

Prompt actions for M365 Copilot follow the wizard: action details (name/description drive semantic matching) → create prompt → input/output parameters → test in M365 Copilot → publish.

Authentication in one screen

  • User authentication to the agent: Authenticate with Microsoft (default; required for Teams + M365 Copilot channel; no User.AccessToken), Authenticate manually (Entra ID/generic OAuth2 — federated credentials recommended over secrets; exposes User.AccessToken), No authentication (public link; can be blocked by DLP; tools can't use user credentials).
  • Agent to tools: delegated (user) connections for permission-trimmed retrieval/actions; maker/application connections for fixed-identity automation; on-behalf-of (OBO) underpins SSO.

⚡ Exam focus: typical trap — a scenario needs an action to run as each signed-in user (audit per user, row-level security) → tool must use user authentication, agent must not be "No authentication." Another: high-volume deterministic step billed cheaply → agent flow (13/100 actions), not generative answers.
