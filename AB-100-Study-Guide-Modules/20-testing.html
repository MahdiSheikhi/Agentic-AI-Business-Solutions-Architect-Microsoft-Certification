Testing Agents & Validating Models

Testing Copilot Studio agents

  • Test panel — walk conversations, see triggered nodes, activity map for generative orchestration (shows the orchestrator's plan and tool selection), live topic editing. Test chats are free and excluded from analytics.
  • Agent evaluation (in-product, preview): test sets up to 100 cases (conversational/multi-turn sets: up to 20; results retained 89 days) — generated from instructions/knowledge/topics, imported, from past test chats, or from analytics Themes; optional simulated user profile. Test methods: General quality (LLM-judged relevance/groundedness/completeness/abstention — no expected answer needed; default) · Compare meaning (semantic vs expected) · Tool use (expected tools/topics invoked — pass/fail) · Keyword match · Text similarity · Exact match · Custom (your evaluation instructions).
  • Copilot Agent Kit (Power CAT; a.k.a. Copilot Studio Kit) — bulk automated testing over the Direct Line API: Response match, Adaptive Card/attachment checks, Topic match (needs transcript enrichment), Generative answers (LLM compares to sample answer/validation instructions; App Insights enrichment explains failures), Multi-turn (ordered child tests = end-to-end scenarios), Plan validation (asserts the orchestrator planned the expected tools). Results include success rate + latency; supports rubrics, Excel import/export, and runs inside Power Platform pipelines (gate promotion on passing tests). The Kit also ships Agent Review Tool (anti-pattern detection), Conversation Analyzer, Prompt Advisor, Agent Inventory, Insights Hub.
  • Strategy layers (per guidance): unit tests → happy path → knowledge tests → regression → adversarial (edge cases, unknown intent, false info) → performance/load → security & compliance (RBAC, DLP, labels, leaks) → accessibility. Shift-left; always validate in staging before go-live; automate in CI/CD.

Testing in Foundry (pre-deployment)

  • Evaluations: local (SDK), cloud at scale, or portal; evaluator catalog with RBAC. Agent evaluators: Intent Resolution, Task Adherence, Tool Call Accuracy (+ task completion, tool-selection/input/output variants). RAG: Groundedness (+Pro), Relevance, Retrieval, Response Completeness. Safety: harm categories, protected materials, XPIA, code vulnerability. Scored 1–5 vs threshold (default 3) or pass/fail; run in CI/CD via GitHub Actions / Azure DevOps with baseline comparison and statistical significance.
  • AI Red Teaming Agent (PyRIT-based) — pre-launch adversarial scans producing an Attack Success Rate scorecard (see the Responsible AI module).

Validating custom AI models (direct exam bullet)

Model Validation criteria
AI Builder prediction Grade A–D (A best; D = worse than random or suspiciously ~100% → data leakage/overfit); accuracy on a held-out test set; numerical prediction graded on R²; inspect the confusion matrix & most-influential columns
Category classification / entity extraction Performance score from precision, recall, F1 + quick test
Document processing Per-field/table accuracy score + improvement recommendations (accuracy shown for fixed-template models)
Fine-tuned models (Foundry) Training curves: train_loss ↓, train_mean_token_accuracy ↑, plus valid_loss / full_valid_* per epoch; divergence between training and validation = overfitting → fewer epochs / lower LR multiplier; automatic safety evaluation (preview) can block deployment; then run Foundry evaluations vs expected pairs; deploy to a Developer deployment first; use the same system message as in training at inference

Threshold philosophy (documented): there is no absolute pass mark — optimize precision when false positives are costly, recall when false negatives are costly, F1 for balance; the acceptance decision is a business call.

Validating prompt best practices (direct exam bullet)

Validate prompts against the four-element structure (goal, context, source, expectations) plus practices: specific wording, action verbs, steps for complex asks, iterate-and-resubmit. Mechanisms: Copilot Prompt Gallery (discover/save/share), Prompt Coach agent template, AI Builder "Test prompt" with real sample data + prompt evaluations/accuracy scoring (preview), Copilot Agent Kit Prompt Advisor, Copilot Studio Optimize prompts guidance.

End-to-end tests across multiple Dynamics 365 apps (direct exam bullet)

Surface Tooling
Canvas apps Test Studio; Power Apps Test Engine (preview — YAML Power Fx tests via PAC CLI; canvas + model-driven; extensible, Playwright-based) and Microsoft's Playwright samples (current direction)
Model-driven / custom pages Test Engine / Playwright samples
Finance & operations apps RSAT — record with Task recorder → automated UAT suites, Excel-parameterized, integrated with Azure DevOps Test Plans. Deprecation: unsupported after May 15, 2027 — plan successors
Agents Test panel + agent evaluation + Agent Kit multi-turn/pipeline runs

Non-deterministic AI changes the method: avoid exact-match assertions → use LLM-judged comparisons, rubrics, thresholds/pass rates, repeated runs; assert plans/tools chosen rather than exact wording; multi-turn tests for conversational E2E; orchestrate cross-app suites (RSAT + Test Engine + agent tests) under one Azure DevOps test plan for traceability; include adversarial and load tests in UAT; verify DLP/RBAC/labels in staging.

Creating test cases with Copilot (direct exam bullet)

Documented AI-assisted generation: Copilot Studio agent evaluation generates test sets from instructions/knowledge/topics and from analytics Themes (production telemetry → regression tests); Agent Kit bulk creation via Excel; Azure DevOps Test Plans via the Azure DevOps MCP server + Copilot ("create test cases for…", coverage-gap analysis); GitHub Copilot test generation for code. Architect guidance: AI generates breadth → humans review; LLM-judged methods for open-ended answers, deterministic methods (exact/keyword/tool-use) for compliance-critical paths; store cases in Azure DevOps; regenerate when knowledge or instructions change.

⚡ Exam focus: map need→method instantly — "regression-test 500 Q&A pairs before each release" → Agent Kit / agent evaluation in pipelines; "prove the orchestrator picks the right tools" → plan validation / Tool use method; "model predicts 100% accuracy" → data leakage (grade D); "F&O UAT automation" → RSAT (note its 2027 sunset).
