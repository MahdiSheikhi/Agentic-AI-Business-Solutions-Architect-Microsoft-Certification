Domain 2 · Design

Copilot Studio III — MCP, Computer Use, Reasoning, Voice, Channels

MCP in Copilot Studio (exam bullet: "agent extensibility with MCP")

  • Supports MCP tools and resources — not prompts (resources surface only as tool outputs). Requires generative orchestration.
  • One MCP server = one agent tool exposing many MCP tools; the server's tool list updates dynamically — no connector redeployment when the server changes. That's the core value vs connectors: centrally versioned, frequently changing tool sets shared by many agents.
  • Transport: Streamable HTTP only (SSE unsupported after August 2025).
  • Add via the MCP onboarding wizard (auth: None, API key header/query, OAuth 2.0 — dynamic discovery/DCR, dynamic, or manual), via a custom connector, or register BYO servers through Agent 365 / M365 admin center. Prebuilt Microsoft MCP connectors exist (Dataverse MCP, Dynamics 365 ERP MCP, D365 Customer Service MCP…).
  • Governance: MCP rides on Power Platform connectors → DLP data policies apply; per-tool enable/disable via the Allow all toggle (when off, new server tools default off). Limitations: can't enrich tool descriptions per agent; topics can't call MCP directly.

Computer use (exam bullet: "automate tasks in apps and websites")

  • A tool that drives websites and Windows desktop apps with a virtual mouse/keyboard, powered by computer-using agent (CUA) models. Loop: Perception (screenshots) → Reasoning → Action; self-corrects when the UI changes. Requires generative orchestration; authored in natural language.
  • Where it runs: hosted browser (preview, on Windows 365 for Agents — prototyping only, not Entra-joined) · Cloud PC pool (preview — Windows 365 for Agents machines that ARE Entra-joined/Intune-enrolled, with autoscaling) · bring-your-own machine for production (Windows machine registered in Power Automate, "Enable for computer use"; Entra-joined/Intune-managed; concurrent runs queue).
  • Models: OpenAI CUA (GA) and Anthropic Claude models (admin must enable "external models" for Anthropic).
  • Controls: app/website allow list, Enforce HTTPS, maker vs end-user credentials, human supervision (email a reviewer on suspicious instructions), session replays/logs.
  • Billing: 5 credits/step (15 premium); excluded from M365 Copilot license inclusion.
  • When to propose: only when no API, connector, or MCP integration exists (legacy/desktop-only UI). Prefer connectors/REST/agent flows — cheaper, deterministic, lower risk. Contrast with classic RPA: RPA for stable UI + fixed rules + high volume; computer use for shifting UIs + judgment.

Deep reasoning & model choice

  • Deep reasoning (preview) enables reasoning models (currently Azure OpenAI o3) for complex multistep analysis. Requires generative orchestration + settings toggle; force per-step with the keyword reason in instructions. Trade-offs: better analysis, slower, extra cost (premium AI-tools token meter plus the agent-action rate). US/EU regions; no data-residency commitments while in preview.
  • A separate primary model picker lets you swap the agent's main model (tags Deep/Auto/General; optional external models — Anthropic, xAI, Mistral — gated by an admin switch), plus BYO Foundry model for prompts.

Voice

  • Basic voice agents — classic orchestration + NLU/NLU+; full IVR feature set: speech + DTMF, silence detection, barge-in, SSML, audio playback, latency messages. Deterministic and low-latency → regulated IVR flows.
  • Real-time voice agents (preview) — speech-to-speech on a real-time multimodal model; natural interruption, mid-call language switching; deployed through Dynamics 365 Contact Center telephony; some triggers (unknown-intent fallback, disambiguation) unsupported.
  • Voice billing per minute: classic 10 / genAI 35 / premium 75 credits. Bot-handled calls limit: 1 hour; context passes to external telephony via SIP UUI / SIP X-headers.

Publishing, channels & sharing

  • Publish once → all connected channels update; republish after every change. Channels: Teams + Microsoft 365 Copilot (requires Authenticate with Microsoft), SharePoint, demo website (stakeholder testing only) vs custom website (embed), mobile/native via Client SDK, WhatsApp/Facebook, Dynamics 365 Customer Service / Contact Center (omnichannel + voice), telephony (Azure Communication Services), Azure Bot Service channels (Slack, Telegram, Twilio, email…).
  • Sharing: co-author or view; Analytics Viewer role for analytics-only; Bot Transcript Viewer for transcripts; org-catalog publishing in Teams needs admin approval; PPAC sharing limits apply (managed environments).

⚡ Exam focus: requirement→feature mapping — "third-party system already exposes an MCP server" → add MCP tool (generative orchestration). "Legacy Windows app, no API" → computer use on a bring-your-own machine for production. "Complex analysis step" → deep reasoning with reason keyword. "Phone menu with DTMF + regulatory wording" → classic voice + NLU(+). "Natural conversation over phone" → real-time voice via Contact Center.
