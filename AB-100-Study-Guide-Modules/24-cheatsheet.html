Naming decoder

Microsoft Foundry ← Azure AI Foundry ← Azure AI Studio · Foundry Tools ← Azure AI services · Foundry Agent Service on the Responses API (Foundry classic agents retire 2027-03-31; Azure OpenAI Assistants API retires 2026-08-26) · Copilot Credits ← messages (renamed 2025-09-01) · Agent Framework ← Semantic Kernel + AutoGen · Copilot Service admin center ← Customer Service admin center · Agent 365 = agent control plane · Foundry IQ / Fabric IQ / Work IQ = knowledge layers.

Platform picker one-liners

  • Prebuilt SaaS agent meets the need → configure, don't build.
  • Team Q&A over M365 content, no-code → agent builder / SharePoint agent.
  • Department/org business process, low-code, DLP-governed → Copilot Studio.
  • Model choice, custom orchestration, evals, CI/CD → Foundry (prompt vs hosted agents).
  • Full code control of multi-agent graphs → Agent Framework (Sequential · Concurrent · Group chat · Handoff · Magentic).
  • Agent↔tools = MCP · agent↔agent across boundaries = A2A · same team = child agent.
  • Legacy UI, no API → computer use (prod = bring-your-own machine). Stable UI + fixed rules + volume → RPA.

Deployment & residency

Data at rest always stays in the resource geography. Processing: Global (anywhere) · Data Zone (US/EU/APAC) · Regional. Batch = ~50% off, 24 h. PTU = reserved throughput (fungible across Azure Direct models; spillover → standard). Developer tier = fine-tune evals, no SLA. Model router: Balanced/Cost/Quality; context = smallest underlying model; Claude targets deployed separately. Cross-geo generative-AI consent = per-environment PPAC toggle. Grounding with Bing leaves the compliance boundary.

Security rapid-fire

  • UPIA = user-typed jailbreak → Prompt Shields (user); XPIA = instructions hidden in content → Prompt Shields (documents) + spotlighting + least privilege + human-in-the-loop; Purview flag XPIADetected.
  • Entra Agent ID: blueprint = template holding credentials; agent identity = credential-less service principal. Attended = OBO; autonomous = client credentials.
  • Foundry RBAC: fine-tune training = Foundry User (data plane); deploying a fine-tune is control-plane — Foundry Owner covers both, or pair User with Account Owner. CMK revocation: no new training/deployments, deployed models keep serving.
  • Filters off/annotate-only = Limited Access approval. Protected material detection on by default.
  • Purview: CopilotInteraction audit events; DSPM for AI = posture + oversharing assessments; Defender for Cloud AI threat protection = runtime alerts.

ALM rapid-fire

Solutions (managed → test/prod) + environment variables + connection references + pipelines (managed-environment targets; no cross-tenant). Not solution-aware: App Insights settings · manual auth · web-channel security · channels · sharing. Topic name with a period blocks export. Component collections = reusable topic/tool sets. Foundry: agents-as-code, azd pipeline config, evaluation GitHub Action as a release gate. Custom models = MLOps/GenAIOps + progressive exposure + drift monitoring.

Analytics KPI definitions (one line each)

Engaged = custom topic / boosting / escalate / fallback triggered · Resolved = confirmed (user says yes) or implied · Escalation reasons: system intended / system unintended / user requested · Abandon = engaged, unresolved, timed out · CSAT = 1–5 survey. Foundry agent evaluators: Intent Resolution · Task Adherence · Tool Call Accuracy. Red teaming output = Attack Success Rate (ASR).

D365 rapid-fire

Business terms = glossary/synonyms in Copilot Studio (SalesSpecificQnA) — shared with the M365 Sales agent · Case summary needs ≥50 tokens of data · Contact Center: standalone (keep CRM) vs embedded; intent-driven unified routing via Customer Intent Agent · handoff passes context variables; voice context via SIP UUI/X-headers · F&O: prereq Power Platform Integration; in-app help extends via files on the "Copilot for finance and operations apps" agent (SharePoint → custom topic; virtual entities not supported for help); chat-with-data via mserp_ virtual entities shared across in-app, custom, and M365 agents · F&O agents deploy via PPAC agent deployment wizard with a dedicated agent identity user · M365 agents: Researcher (research) · Analyst (data) · Facilitator (meetings) · SharePoint agents (site Q&A).

Responsible AI

Six principles: fairness · reliability & safety · privacy & security · inclusiveness · transparency · accountability (Standard v2 = 14 goals → requirements; impact assessments + transparency notes). Shared responsibility: SaaS Microsoft ↔ PaaS you own system message/grounding/filters. Review = principle-by-principle checks + named owner + human oversight + audit trail + release-gated evals/red teaming.
