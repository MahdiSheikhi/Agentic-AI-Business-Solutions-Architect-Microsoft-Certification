Domain 3 · Deploy

Monitoring Agents & Interpreting Telemetry

Deploy is 40–45% of the exam, and monitoring is its beating heart. Learn the KPI definitions exactly.

Copilot Studio analytics (per agent)

Data retained 360 days (session details/transcripts 28 days in-product; timestamps UTC; test-panel traffic excluded). Views adapt: conversational, autonomous (run outcomes, trigger/tool/knowledge use, Savings estimates), or hybrid.

KPI Definition
Total sessions Count of analytics sessions (one conversation can spawn several; a session runs up to 60 minutes / 100 turns)
Engagement rate % of sessions that trigger a custom topic or enter Conversational boosting / Escalate / Fallback (generative: plan includes a non-system topic, knowledge, or tool)
Resolution rate % of engaged sessions resolved — confirmed (user says yes to "Did that answer your question?") or implied (ends without escalation and logic implies success)
Escalation rate % of engaged sessions escalated; reasons: System intended / System unintended / User requested
Abandon rate Engaged sessions ending unresolved & unescalated after the 60-minute window
CSAT Average 1–5 from the End of Conversation survey

Also: Reactions (thumbs up/down + comments; 28-day retention; not on the M365 Copilot channel) · Themes (AI-clustered questions — convertible into evaluation test sets in one click) · Generated answer rate & quality (AI labels sampled answers Good/Poor for completeness, relevance, groundedness, abstention) · knowledge-source and tool use/error panels · AI-generated insights (preview; daily coverage recommendations) · custom metrics (up to three natural-language business metrics).

Transcripts: stored in Dataverse ConversationTranscript (default 30-day bulk-delete; extendable); viewing needs Bot Transcript Viewer; export long-term via Azure Synapse Link → ADLS Gen2 for custom analytics.

Application Insights connection: paste the connection string in Settings → Advanced. Sends messages/events, topic triggers, node execution, custom telemetry events; options to log conversation details and sensitive properties; KQL over customEvents finds content-filter (ContentFiltered) events. The Power CAT Copilot Agent Kit — Agent Insights Hub merges App Insights + transcripts + PPAC credit usage into one dashboard.

Foundry observability

  • Tracing — OpenTelemetry GenAI conventions into Application Insights (spans per model call, tool invocation, agent decision); supported for Agent Framework, LangChain/LangGraph, OpenAI Agents SDK.
  • Agent monitoring dashboard (preview) — token usage, latency (>10 s = investigate), run success rate (<95% = investigate), evaluation scores, red-team results.
  • Continuous evaluation — evaluators run on sampled production responses (rate-limited ~100/hour default); scheduled evaluations against benchmarks; scheduled red-team scans; alerts on latency/tokens/eval scores. Evaluation run comparison applies statistical significance (Improved/Degraded/Inconclusive) — the documented way to justify a model swap.

Interpreting telemetry → action (the exam's favorite table)

Signal Architect's action
Low engagement rate Coverage gap or unclear entry points → add topics/trigger phrases, clarify description/greeting
Unanswered questions / low generated-answer rate Add/expand knowledge sources; act on AI-generated insights
Poor answers labeled ungrounded/irrelevant/incomplete Tune knowledge content, instructions, topic config; consider stricter grounding (e.g., Bing Custom Search scoping)
Thumbs-down / low CSAT / negative sentiment Drill into transcripts (Bot Transcript Viewer); fix offending topic/knowledge; prioritize by Themes volume
High escalation (System unintended) Users getting stuck — fix topic flows; compare with Omnichannel deflection analytics (they compute differently!)
Knowledge-source errors Repair/replace failing sources
Content-filter blocks KQL ContentFiltered events + transcripts → adjust moderation level or user guidance
Foundry: latency >10 s / high tokens / success <95% Trace the bottleneck span; shorten prompts; check throttling; investigate failed runs
Everything prompt-tuning can't fix Compare models via evaluations → swap model or fine-tune, validated by run comparison

Continuous-improvement loop: assess usage → define outcomes → analytics strategy → data-driven optimization on a cadence (monthly optimization review: deflection, resolution, low-resolution topics, unrecognized utterances, per-channel). Feed Reactions/CSAT/Themes/unanswered lists into a prioritized backlog; convert every fix into a regression test set.

⚡ Exam focus: engagement→resolution→escalation→abandon definitions; resolved-confirmed vs resolved-implied; reactions retained 28 days; where transcripts live and who can read them; >10 s latency and <95% success thresholds; which dashboard answers which stakeholder.
