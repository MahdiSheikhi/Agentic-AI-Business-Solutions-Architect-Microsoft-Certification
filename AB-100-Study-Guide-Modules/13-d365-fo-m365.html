Domain 2 · Design

Dynamics 365 F&SCM, Microsoft 365 Copilot for Sales/Service & M365 Agents

Finance & Supply Chain (finance and operations apps)

Copilot experience types: sidecar (in-app chat panel), embedded (AI inside pages — e.g., confirmed POs with changes, collections summaries, demand-planning analysis), outside (external agents over F&O data). Plumbing: the AI chat control + Copilot Studio as orchestration hub + Dataverse. Prereqs: Power Platform Integration enabled per environment; tenant setting "Publish bots with AI features"; per-feature enablement in feature management. Newer (production-ready preview): Immersive Home (work with agents + prioritized work items), Agent management in-app, Dynamics 365 ERP MCP server (data ops + business logic without custom code).

In-app help & guidance (exam bullet): the sidecar Q&A is grounded in public Microsoft Learn docs by default. To add your own knowledge: Copilot Studio → the environment's "Copilot for finance and operations apps" agent → Knowledge → upload files (PDF/RTF/Word) → test → Publish; answers flow through the built-in conversational boosting topic. SharePoint or other source types require creating your own topic in that agent. Dataverse virtual entities are NOT supported as knowledge for generative help. Optional AI general knowledge (model + Bing) is used only after custom sources are exhausted.

Agent-chat interoperability (exam bullet): expose F&O structured data via virtual entities (mserp_*, e.g., mserp_inventoryonhandaientity) or dual-write tables → add as Dataverse knowledge to (a) the F&O in-app agent, (b) custom Copilot Studio agents, (c) declarative agents in M365 Copilot. Same knowledge, three surfaces, always respecting user data access.

F&O-side first-party agents (deployed via the agent deployment wizard in PPAC Copilot hub — prereq check, refresh virtual entities, select a dedicated agent identity user with roles in both Dataverse and the F&O app, create connection references, activate flows, publish):

Agent Purpose
Account Reconciliation Agent (Finance) Proactive subledger↔GL matching, exception flagging, suggested/executed resolutions
Supplier Communications Agent (SCM) Automates supplier follow-ups, PO updates/change requests per rules
Expense / Time / Approvals agents (Project Operations) Receipt-to-expense-report automation · time entry · policy-based approval classification (Classify vs Classify and auto approve)
Sales Order Agent / Payables Agent (Business Central) Captures sales orders from customer emails · drafts vendor invoices from a monitored mailbox

Microsoft 365 Copilot for Sales & Copilot for Service (exam bullet: orchestrate their configuration)

  • Sales agent (Copilot for Sales): seller assistant in Outlook, Teams, M365 Copilot Chat connected to Dynamics 365 Sales or Salesforce. Deploy: M365 admin center integrated apps (fixed/auto-deployed), Teams setup policy to pin, enable transcription for meeting summaries; each user needs an M365 Copilot license. Configure glossary (shared with D365 Sales Copilot), summary fields. Extensible via Copilot Studio connector actions + partner insights + enrichment APIs.
  • Copilot for Service: generative AI for service reps without rip-and-replace — grounds on existing knowledge (websites, SharePoint, KBs, files) with out-of-the-box integrations for Salesforce, ServiceNow, Zendesk + Dynamics 365; experiences in Outlook/Teams (case summaries, drafted replies, update CRM records); "Extend with Copilot Studio" for topics/actions.
  • Related: Service Agent — M365 Copilot agent for D365 Customer Service reps (prioritize cases, summarize, retrieve knowledge, case write actions).

Microsoft 365 agents for business scenarios (exam bullet: propose them)

Agent Propose for
Researcher Deep multi-step research over web + work content; cited, shareable reports (first-party, appears under Copilot Chat "Tools"; auto-available to licensed users)
Analyst Advanced data analysis over datasets/spreadsheets (first-party, like Researcher)
Facilitator (Teams) Real-time collaborative meeting notes, in-meeting Q&A; notes visible to all participants
Channel Agent (Teams) Channel collaboration: deadlines, status summaries, task assignment
SharePoint agents Site/library-scoped Q&A, created by end users, permission-trimmed; one "approved" agent per site
Sales agent / Sales Chat CRM-connected selling in the flow of work (see above)
Planner agent / Project Manager Plan creation and task execution in Planner (premium)
Skill coaches (Prompt Coach, Idea Coach, Writing Coach…) Prompt/skill upskilling scenarios

Governance: M365 admin center (enable/disable/assign/block), Agent Store, and Microsoft Agent 365 as the cross-origin control plane.

⚡ Exam focus: F&O questions hinge on the plumbing prerequisites (Power Platform Integration, feature management, agent identity user with dual roles) and the knowledge rules (files → default boosting topic; SharePoint → custom topic; virtual entities → chat with data, not generative help). M365-agent questions are matching exercises — research→Researcher, data analysis→Analyst, meetings→Facilitator, site Q&A→SharePoint agent.
