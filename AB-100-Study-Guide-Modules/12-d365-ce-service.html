Dynamics 365 — Customer Experience, Service & Contact Center

Copilot in Dynamics 365 Sales

Enablement chain (architect-level): PPAC Copilot hub → turn on AI for D365 Sales → consent to cross-region data movement where no local Azure OpenAI endpoint → Sales Hub App Settings → Copilot (global + per-app) → enable auditing on lead/opportunity/account tables (required for "recent changes"). DLP must allow required connectors.

Business terms (direct exam bullet) — configured in Copilot Studio, not the Sales app: environment → agent "Copilot in Dynamics 365 Sales" → Knowledge → SalesSpecificQnA → Glossary (term + description) and Synonyms (alternate column names) → Publish. Use for acronyms, custom fields, custom ownership models, complex rules ("overdue task" = open task with due date < today). Takes ~15 minutes to apply. The same glossary serves Copilot in D365 Sales and the Sales agent in Microsoft 365 Copilot (Salesforce-connected: glossary rows in the Dataverse CopilotGlossary table; synonyms unsupported).

Customizations: custom topics via Copilot Studio (needs Copilot Studio Author role; billed as Copilot Studio credits; use Adaptive Cards, not images/video); edit the SalesSparks prompt-guide topic; configure summary fields for lead/opportunity/account summaries; add the opportunity summary widget / Copilot immersive page to custom forms & site maps. Manage customizations in solutions for ALM.

Connectors for Copilot in D365 Sales (exam bullet): (1) custom connector + connector action built in Copilot Studio, enabled by a Sales admin (ingestion up to 7 days); (2) partner-application insights via Power Platform connectors (e.g., DocuSign records/activities shown in context). Both bring external data into seller experiences.

Copilot in Customer Service (admin = Copilot Service admin center)

Prereq: PPAC "Enable AI-powered Copilot features" per environment. Then configure under Productivity:

  • Copilot help pane — Ask a question (knowledge base + trusted webpages + optional Bing web search — non-NA regions need cross-geo consent), Write an email, draft chat responses; Copilot immersive vs side pane.
  • Summaries — Case summary (requires ≥50 tokens ≈ 38 words of case data; Manage data remaps/includes/excludes source fields; exclude up to 10 email addresses, 3 disclaimers) and Conversation summary (triggers: rep joins/transfer/collaborator, conversation end — optional Create case button — and on-demand; Manage format).
  • Access via experience profiles — enable features per representative group; profiles are solution-aware (ALM-able).
  • Case-form summaries are Dynamics 365 Customer Service-only; in Contact Center—embedded (non-Microsoft CRM) a case-summary card appears in the Ask a question pane (preview). Row-summary control is moving to PPAC Copilot hub.

Customer Insights: Dialog with data (preview, NL questions over unified profiles), Copilot-assisted segment creation, journeys copilots (query assist, content ideas). Enable via the global consent toggle.

Dynamics 365 Contact Center

Microsoft's CCaaS. Deployment scenarios (know all three): Contact Center—standalone — the Microsoft-provided rep workspace with the CRM of your choice (unified routing incorporates non-Microsoft CRMs like Salesforce/ServiceNow) · Contact Center—embedded — the Contact Center experience embedded inside your existing non-Microsoft CRM's UI · with Dynamics 365 Customer Service — the full Microsoft stack (= Customer Service Premium). Channels: voice (PSTN via Azure Communication Services, or bring-your-own-carrier/direct routing), chat, SMS/social, email, Teams, custom via Direct Line, third-party telephony via Channel Integration Framework 2.0.

Unified routing = two stages: classification (rules + ML: skills, urgency, intent) → assignment (priority + skills/capacity/availability matching). Supports percent-based routing, overflow, preferred-rep, and intent-driven routing (Customer Intent Agent intents mapped to queues).

Copilot Studio agents in Contact Center (exam bullet): create in Copilot Service admin center or Copilot Studio → connect to the omnichannel instance → add to a workstream ahead of human queues. Voice/IVR: DTMF + speech, silence detection, SSML, barge-in, external speech grammars, transfer to rep/external number, multilingual; 1-hour bot-call limit; SIP UUI / X-headers for external telephony context. Handoff with context: escalation passes full history + context variables (drive routing rules, macros, record linking — e.g., pass case ID); rep availability APIs let the agent check queues before escalating; install the D365 Copilot Studio extension (+ telephony extension). Voice handoff: both environments must be in supported geos; auth variables don't pass on voice handoff.

First-party agents (CE & service side)

Agent Purpose / notes
Case Management Agent (Customer Service) Automates case lifecycle: create/update/resolve/follow-up/close
Customer Knowledge Management Agent Autonomously harvests knowledge articles from closed cases/conversations; scrubs sensitive data; needs knowledge management configured + connection references + flows + published agent
Customer Intent Agent (+ voice) Mines past cases into an intent library; guides reps; powers self-service + intent-based routing; needs Intent Manager role
Quality Evaluation Agent Autonomous quality assessments against supervisor-defined frameworks
Customer Assist / Quality Assurance / Service Operations (platform agents) Self-service + rep assist / real-time QA nudges / conversational ops-config
Sales Qualification Agent Researches leads (CRM + web + knowledge), outreach, engages, hands over qualified leads; setup: Azure app registration → Dataverse application user with AISalesPerson role; Research and engage mode needs a shared mailbox + server-side sync; one mode per org; BANT handoff criteria
Sales Opportunity / Sales Close / Sales Research / Recommended Actions agents Opportunity risk research · autonomous close cycle · NL data dialog · next-best actions

Customer Service agents require pay-as-you-go (consumption) billing to be set up.

⚡ Exam focus: "Copilot misunderstands our org's terminology in Sales" → glossary/synonyms in Copilot Studio (SalesSpecificQnA). "Keep Salesforce but modernize the contact center" → Contact Center standalone (Microsoft rep workspace) or embedded (inside the Salesforce UI); full Microsoft stack → Customer Service Premium. "Route by what the customer wants" → Customer Intent Agent + intent-driven unified routing. "Case context must reach the human rep" → context variables in the handoff.
