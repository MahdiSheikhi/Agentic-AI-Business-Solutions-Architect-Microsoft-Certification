Power Platform AI — AI Builder, Canvas Apps, Generative Pages, Well-Architected

AI hub & AI Builder

  • AI hub lives in the Power Apps and Power Automate maker portals ("… More → AI hub") — the entry point to AI models and Prompts (exam bullet: "Propose Microsoft Power Platform AI features, including AI hub").
  • AI Builder (requires Dataverse; enable per environment): prebuilt models (receipt/invoice/business card processing, text recognition, sentiment…) and custom models (document processing, prediction, category classification, entity extraction, object detection). Consumed from Power Apps, Power Automate, and agents; licensed via AI Builder credits; admins can disable preview models per environment.
  • Prompts (prompt builder): generative prompts on GPT-class models; grounded in Dataverse; multimodal inputs; solution-aware and reusable across copilots, apps (Power Fx), flows, Dataverse. (Details in the Copilot Studio tools module.)
  • Plan designer (Plans, GA): "copilot-first development" — describe the business problem (plus process images) → generates roles, user stories, data model, and the solution set: Dataverse tables, canvas + model-driven apps, Power Pages, flows, Copilot Studio agents, Power BI; can generate a plan from an existing solution; the plan doubles as documentation. Propose for greenfield envisioning and requirements-to-solution acceleration.

AI components in canvas apps (direct exam bullet)

Mechanism Notes
Copilot control (preview) Chat over the app's Dataverse data; needs PPAC consent setting + app settings. Deprecation signal: cannot be added to new canvas apps from Feb 2, 2026 — Microsoft 365 Copilot in apps is the successor
Add a custom Copilot Studio agent (preview) Embed an existing agent; works web and mobile; the modern answer for in-app assistance
AI Builder prompts as Power Fx functions Add the prompt as a data source → 'PromptName'.Predict(input) → bind result.text. The primary pattern for embedding generative steps (classify, extract, draft) directly in a canvas business process
Model-driven counterparts App Copilot chat, row summaries, form fill assistance, NL view filtering

Code-first generative pages & the agent feed (direct exam bullet)

  • Generative pages (model-driven apps; GA Oct 30, 2025): describe a page in natural language (+ up to 6 Dataverse tables, reference images) → generates React/TypeScript code, refined conversationally. Two approaches: (1) maker-portal conversational designer (preview regions: US, GB, AU, SG); (2) code-first with external AI tools (GitHub Copilot CLI, Claude Code) — generate/edit locally against Dataverse, push via CLI; pushed pages behave as native generative pages. Solution-aware ("UX Agent Project"). Propose when: pro-dev teams want frontier models/local tooling, unsupported regions, or pixel-level custom UX without full PCF development. Makers must validate generated code.
  • Agent feed for apps (preview): surfaces agent-generated actionable tasks inside model-driven apps — the human-supervision surface for autonomous agents. Sections Needs attention (request_assistance, invoke_data_entry) and Completed (request_review); requires agents onboarded to the Power Apps MCP server (only MCP-server agents appear from May 1, 2026). Propose when autonomous agents create/update Dataverse records needing human review inside the operational app. Caution: everyone with access to the Agent Task table sees feed items.

Power Platform Well-Architected (direct exam bullet)

Five pillars — four align to Azure WAF, one is unique:

Pillar Concern
Reliability Resiliency, availability, recovery
Security Data protection, threat detection
Operational Excellence Observability, fusion-team DevOps, safe deployment
Performance Efficiency Scalability under demand
Experience Optimization (unique to PPWAF) Usability, relevance — includes conversational UX design for agents

Each pillar: design principles → checklist → recommendation guides with explicit trade-offs; the Well-Architected Review assessment produces prioritized recommendations (~90-day iteration cadence). Intelligent Application Workloads guidance (added Sept 2024) applies PPWAF to generative-AI/agent workloads and now lives in the Copilot Studio guidance library (Plan / Implement / Adopt / Manage / Improve / Extend); its security thread: group-based licensing, security roles via group teams, environment-tiered DLP (strict in dev → managed prod), sharing limits, VNet/IP firewall for mission-critical agents, residency and cross-geo controls.

⚡ Exam focus: "include AI in a canvas-app business process" → prompt as Power Fx function (or embed an agent) — not the deprecated Copilot control. "Custom-coded page UX generated by AI, worldwide, pro-dev" → code-first generative pages. "Humans must supervise what autonomous agents did to records" → agent feed. "Which pillar is unique to Power Platform WAF?" → Experience Optimization.
